js中的一些细节

1级

1、XHTML:HTML遵循xml的规则。XHTML中“<” ，“>”需要使用html实体(<&lt;)替代，当然，这样会导致代码不容易理解，可以使用CDATA区域来解决（CDATA区域的语法规则：<![CDATA[ ... ... ]]>）

2、使用引入的方式而不是直接将代码写在html中的有点有3个：1、便于维护，2、可缓存，3、避免xhtml以及其他xml技术的不兼容。

3、文档模式（<!DOCTYPE HTML PUBLIC "" "">）的概念：混杂模式，标准模式，会影响css的呈现和js的执行。

4、noscript标签：标签中的内容，会在js功能缺失的情况下执行

2级

1、为初始化的变量初值是undefined

2、可以在修改变量值的同时，修改变量的类型 var message = "hello"; message = 100;

3、var关键字控制的局部变量和全局变量 function add(){ var a ="a-test"; alert(a); } 以上的代码，因为使用了var来声明a，所以，a是局部变量，当方法返回后，a便会被销毁

4、typeof是操作符而非函数，我们会看到 typeof (95)这样的用法，其中"95"两边的括号可以不要，两种语法都可以正常使用。 typeof的返回值是字符串，有6个。 "undefined"——>如果测试值未定义 "boolean" "string" "number" "object"——>测试值是对象或null，也就是说 typeof null 返回的是字符串"object" "function"——>测试值是函数

5、数值类型的值可以是 十，八，十六进制值，

6、NaN:NaN（Not a Number）,是一个特殊的数值，非数值。当一个本来要返回数值的操作，未返回数值的时候，会返回此值，这样避免了抛出错误。 如666/0，会返回NaN,而不是抛出错误，不会影响其他代码的执行。 NaN两个不同寻常的特点：1、任何设计NaN的操作都会返回NaN,当然这是游客嫩导致问题的。2、NaN与任何值都不想等，包括NaN本身。NaN == NaN返回的是false。 isNaN(Param)方法：返回的是true和false，当传入的Param可以转换成数值的时候，返回true，否则返回false。 isNaN方法也使用与对象。

7、isFinite(Param):当Param的返回在Number.MAX_VALUE和Number.MIN_VALULE之间时返回true，否则返回false。

9、Number.NEGATIVE_INFINITY,Number.POSITIVE_INFINITY
